# Apple-Style Response Guide

## Hierarchy Levels

### Primary Action (Top-level message)
Use **semibold** for the main message or accomplishment.
Single line, direct.

**Example:** **Favicon updated and deployed.**

---

## Secondary Information
Use regular weight text below the primary action.
One sentence max per point.

---

### Details Section
Use indented structure for related items.
Format as:
- **Label:** Value
- **Label:** Value

**Example:**
- **Status:** Complete
- **Size:** 192x192px
- **Location:** /public/favicon.png

---

## Actions/Steps
When listing steps or actions, use:

**Action name**
Description of what was done.

---

## Color Usage
- **Semibold text** = emphasis, completion, primary info
- *Regular text* = secondary details
- Inline code for file paths, commands: `/path/to/file`

---

## Spacing & Structure
- Line breaks between major sections
- No unnecessary borders or visual noise
- Grouped related items together
- Use em dashes sparingly; prefer periods or line breaks

---

## Example Full Response

**Setup complete.**

Your checklist app is running with the new circular favicon. The blue checkmark displays at 192x192px.

- **Status:** Deployed to both /public and /dist
- **URL:** http://localhost:5174/
- **Changes:** Circular edges, increased size

The favicon is now live in the browser tab.

---

## When to Use
Every response should follow this structure:
1. **Bold statement** of what was accomplished
2. Supporting details in hierarchy
3. Grouped information with labels
4. Minimal visual clutter
5. Clear, direct language
